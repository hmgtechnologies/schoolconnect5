## 0. 🐞 Builder wizard "Next: Branding" does nothing (client-reported)

**Symptom:** on the builder, filling Step 1 and clicking **Next: Branding** did nothing.

**Root cause (in the older live build):** `builder.html`'s inline script rendered the
theme/font/layout grids at the **top** of the block, *before* `Wizard.init()` ran at the
bottom. If any of those render lines threw (e.g. `document.getElementById('theme-grid')`
was null, or `window.SC` wasn't ready) the **whole inline script aborted**, so
`Wizard.init()` never ran and **no buttons were ever bound** → clicking Next did nothing.

**Fix (defensive, permanent):**
- `Wizard.init()` (+ PWA + service-worker) now run **first**, before any grid rendering,
  so navigation is bound no matter what.
- The grid-render code is wrapped in **try/catch** (and null-guards `window.SC` /
  elements), so a render failure can never block the wizard.
- `selectTheme/Font/Layout` guard `event.currentTarget`; the logo handler null-guards.
- Removed the duplicate bottom `Wizard.init()` (prevents double-binding / double-step).

**Verified (jsdom, against the real files):** Next advances exactly 1 step (1→2→3),
prev/step-indicators/presets all work, 86 theme cards render — **and even when
`window.SC` is deliberately sabotaged to throw during grid render, Next still advances
to step 2.** `verify.sh` asserts the hardening.

## 2. ✨ Timetable enhancement — part-time teachers

Part-time / visiting teachers attend only specific weekdays. The timetable now respects this.

- **Schema:** `timetable_requirements` gains `available_days text[]` + `is_part_time`;
  a reusable `teacher_availability` roster table is added.
- **Generator (`generate_timetable`):** for each teacher it uses, in priority order,
  the requirement's `available_days` → the `teacher_availability` roster → all weekdays;
  it **only places that teacher's periods on the days they attend**, and reports any
  **`unplaced`** periods if their availability/periods-per-day can't fit the demand.
- **UI (`timetable-generator.html`):** a **"Part-time teacher"** checkbox reveals
  Mon–Fri day checkboxes; the demand table shows each teacher's *Attends* days; generation
  warns if some periods couldn't fit.
- **Verified (PostgreSQL 17):** a Tue/Thu-only teacher got **all periods on Tue/Thu**,
  **0 violations**; an over-constrained teacher correctly reported `unplaced`.

## 3. ✨ School Assistant chatbot — enhanced (still no AI)

- **KB expanded** from 14 → **29 topics** (login, approval, 2FA, CBT, taking exams,
  grading, report cards, fees, attendance, QR check-in, timetable, **part-time**, voting,

  data, analytics, language/accessibility, search, dark mode, cost, deploy, contact).

  "timetable for a visiting teacher" → part-time answer).

- **Quick-reply chips** — tappable suggestions on the greeting and after every answer.
- Friendly greeting/thanks/goodbye handling and a helpful fallback with topic chips.
- 100% rules-based and offline — **no AI API**.
