1️⃣ Quick Start — 4 Steps

Step 1 · Fill the Wizard

  1. Open the builder — go to builder.html or click "Start Building" on the home page.
  2. Pick a preset or fill the form manually. A preset pre-fills every module you need.
  3. Step 2: choose a theme (14) and font (8).
  4. Step 3: pick a layout — Sidebar, Top Nav or Card Hub.
  5. Step 4: list your classes/levels (comma-separated).
  6. Step 5: tick the modules you want (44 available).
  7. Step 6: optional — paste Supabase keys, then click Generate & Download ZIP.

Step 2 · Create the Free Database (Supabase)

  1. Go to supabase.com → start a new project (free).
  2. Wait ~2 minutes for provisioning.
  3. Open SQL Editor → New query.
  4. Open the unzipped database/schema.sql file, copy it, paste, and click Run.
✓ Done? You should see "Success. No rows returned." The schema is idempotent — safe to re-run.

Step 3 · Connect the App

In Supabase, go to Project Settings → API and copy your URL + anon public key. Open the unzipped assets/js/config.js and replace the placeholders:

window.SUPABASE_URL = 'https://abc.supabase.co';
window.SUPABASE_KEY = 'eyJ...your-anon-key...';

Step 4 · Publish (Free)

Upload the unzipped folder to GitHub Pages, Cloudflare Pages, Netlify or Vercel. All are free.

2️⃣ Voting & Polls

Students vote for prefects, head boy/girl, staff polls. Live results, anonymous ballots, multi-channel notifications.

How to create a poll

  1. Sign in as admin / principal / head teacher.
  2. Click Voting & Polls in the sidebar.
  3. Click + New Poll. Fill the form:
    • Title — e.g. "Head Boy Election 2026".
    • Type — single choice, multi choice, yes/no or ranked.
    • Candidates — one per line: Name | Class/Arm | Photo URL.
    • Audience — students, staff, parents or everyone.
    • Anonymous — checked if you want anonymous ballots.
    • Closes at — optional deadline.
  4. Click Create. ✓ All eligible users get a notification instantly.

How voting works

  • Voters see live results — bars update as people vote.
  • One vote per user per poll (DB-enforced). They can change their vote.
  • Anonymous mode: votes are stored but voter_id is hidden in the UI.
  • Closing a poll triggers another notification with the final results link.
Try it: visit voting.html for a live demo.

3️⃣ Multi-Channel Notifications

Every announcement, broadcast, message, poll and result triggers all enabled channels automatically.

📨 In-app bell
🔔 Browser push
📧 Email (free)
💬 WhatsApp (free)
📱 SMS (free)
🌍 Multi-language

How to enable push

  1. Open the school site in Chrome / Edge / Firefox.
  2. Click the install banner at the bottom of any page (or use install.html for help).
  3. After install, click the bell icon — grant notification permission.
💡 Tip: Browser push only works after PWA install. Email, WhatsApp and SMS work everywhere.

How to send a broadcast

  1. Open Messaging in the sidebar.
  2. Pick the audience (class, role, or all).
  3. Compose the message. Pick channels.
  4. Click Send. WhatsApp opens wa.me, email opens mailto:, SMS opens sms:.

4️⃣ PWA Install Enforcement

Every page on the platform shows an install banner until users install the app. This guarantees parents and teachers actually install it (so they receive push notifications).

Detection

  • Chrome / Edge on desktop: native install prompt appears.
  • Android: "Install app" menu option.
  • iOS Safari: step-by-step helper appears automatically.

For admins

You can disable the banner or change its frequency by editing the PWA-install banner block in any generated page.

7️⃣ Security & RLS

  • User-chosen passwords via Supabase Auth (bcrypt-hashed server-side).
  • Row-Level Security on every table (95+ least-privilege policies).
  • Only the anon public key reaches the browser. Never the service-role key.
  • All data rendered into the DOM is HTML-escaped (defence-in-depth).
  • Suspended / pending accounts are blocked at login.
